# 360-Video Caching Optimization via DRL & Greedy Algorithm
**TV 3.0 브로드밴드 환경의 품질 향상 및 백홀 절감을 위한 강화학습 기반 360도 비디오 캐싱 정책 산출 프로그램**

본 저장소는 TV 3.0 환경의 브로드밴드 엣지 서버를 대상으로, 제한된 캐시 용량 내에서 영상 품질(QoE)을 극대화하고 백홀 트래픽을 최소화하기 위한 심층 강화학습(DRL) 및 그리디(Greedy) 알고리즘 기반의 캐싱 최적화 프레임워크를 포함하고 있습니다.

## 📌 주요 모듈 및 기능 (Key Modules & Features)



* **시뮬레이션 워크로드 생성 모듈**: 
    * TV 3.0 브로드밴드 환경을 반영한 360도 비디오 요청 패턴, 인기도 변화, 네트워크 대역폭 및 사용자 FoV(Field of View) 분포 워크로드를 생성합니다.
* **DRL 모듈 (Deep Reinforcement Learning)**: 
    * 각 세그먼트의 타일 비트레이트 버전을 결정하는 의사결정 과정을 MDP로 모델링합니다.
    * 영상 품질 유지와 백홀 트래픽 감소를 동시에 고려하는 '캐싱 게인(Caching Gain)' 보상을 극대화하도록 정책을 학습합니다.
* **그리디 알고리즘 모듈 (Greedy Algorithm)**: 
    * DRL이 산출한 세그먼트별 캐싱 게인 정보를 기반으로 전체 캐시 공간을 최적으로 배분합니다.
    * 이득이 큰 조합부터 순차적으로 선택하는 방식을 통해 시스템 전체의 효율을 극대화합니다.

## 🚀 사용 방법 (Usage)

1.  **환경 준비**:
    * 프로젝트 루트 디렉토리에 `sb3stateinfo` 폴더를 생성합니다.
2.  **워크로드 생성**:
    * `hours24simulator.py` 파일을 실행하여 시뮬레이션에 필요한 가상 워크로드 입력 파일을 생성합니다.
3.  **DRL 학습 및 결정**:
    * `sb3ppo.py` 파일 내 `is_train` 변수를 설정하여 실행합니다.
    * **학습 단계**: `is_train = True`
    * **의사결정 단계**: `is_train = False`
4.  **캐싱 게인 계산**:
    * `expecetqoe.py`를 실행하여 각 세그먼트별 기대 캐싱 게인을 산출합니다.
5.  **최종 캐시 할당**:
    * `seg_storage_greedy.py`를 실행하여 그리디 모듈을 구동하고, 최종 비디오 품질 및 백홀 트래픽 결과를 도출합니다.
